范式一 · 浅色开场页 split hero

第 0 章 · 开场

为什么你的 AI
用起来像 金鱼？

每开一个新对话，它都从零开始：不记得你的项目规范、不记得昨天的决定 —— 你只能反复地教它同一件事。

问题不在模型聪不聪明，在于它没有记忆，也没有唯一真相。

没有记忆系统「我们用 ObjectId 不用 slug」→ 教一次 · 下次又写错 · 再教一次 …
没有唯一真相「这个数到底以哪份为准?」→ 三份文档三个答案 · AI 猜一个给你

大师和新手的差距：
不是 prompt 技巧，是有没有给 AI 建好「记忆 + 真相」。

范式二 · 深色三栏卡 dark canvas

核心概念 #2

每次让 AI 干活，先想清三个词

GOAL目标

剥到最根本：你真正要的结果是什么 —— 可验证，不是「帮我看看」。

例：把这门课大纲同步到 production，且 diff 我确认过
CONTEXT上下文

完成目标需要的信息 —— 当下喂进去的规则、文件、数据。

例：CLAUDE.md 规范 + outline.json + 当前 prod 状态
MEMORY记忆

跨会话持久留下的真相 —— 下次不用重讲。

例：记忆库里「ObjectId 不用 slug」「prod ID 在哪查」
🚀 GOAL = 第一性原理 别问「别人怎么做的」，问「我到底要什么结果」—— 从根上定目标，AI 才不会把错的事做得很快。

大多数人只给 Goal，跳过 Context、根本没建 Memory —— AI 当然只能瞎猜。
